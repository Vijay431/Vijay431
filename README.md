## Open Source Direction

I am especially interested in tools that make AI-heavy repositories easier to maintain: explicit repo contracts, safer generated-file boundaries, repeatable security checks, and better handoff between humans and coding agents.

One proposal I am exploring publicly: [a `.gitallow` allowlist for AI-generated repo artifacts](https://github.com/orgs/community/discussions/200163), intended to reduce accidental commits of temporary agent files and internal development context.
